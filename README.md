# First Chorus

**はじめてのジャズアドリブ**
*Your First Step Into Jazz Improvisation*

First Chorusは、楽器を演奏でき、楽譜も読める人が、コード進行を理解しながら、**自分でジャズのアドリブを作る**ための練習アプリです。

コードトーン、ガイドトーン、ターゲットノート、アプローチノート、リズム、モチーフなどを段階的に学び、「コードを確認 → 使える音とルールを確認 → 伴奏に合わせて自分で演奏 → 自己評価」のサイクルで、自分のフレーズを作る力を身につけることを目指します。

公開URL: https://tanbo-hiroooi.github.io/Jazz-improvisation/

## 想定ユーザー

- 楽器の演奏経験がある(吹奏楽・クラシック・ポップスなど)
- 五線譜が読める
- アドリブは初心者
- スケールは分かるが、フレーズにできない
- コード進行に合わせた音選びを学びたい

## 3つの主要モード

### 自由練習
苦手な技術を自分で選び、反復練習するモード。コードトーン/ガイドトーン/アプローチ/ターゲット/スケールの表示を切り替え、11種のリズムパターンや練習課題(ルール提示型)と組み合わせて練習します。

### 練習コース
理論の暗記からではなく、**リズム→フレーズ→コード→アドリブ**の順で学ぶ「リズムから始めるアドリブ入門」(全6章・23レッスン)を収録。基本ルールは「一度に変えるものは1つだけ」です。

### 曲で実践
覚えた技術を、ジャズブルースや長いコード進行の中で使用するモード。実践ミッション(例:「1コーラス目はコードトーンだけ」)を選び、コーラス周回カウントを見ながら1コーラスのソロに挑戦します。

## 練習コースの特徴

章構成: ①スウィングのノリを覚える(音は1音・リズムだけ) → ②コードトーンでフレーズを作る(4小節を完成) → ③コード進行を音で表現する(ガイドトーン・ターゲット・アプローチ) → ④モチーフを育てる → ⑤ジャズブルースで実践(ブルーノート・リフ・掛け合い) → ⑥1コーラスのソロを作る(12小節)

各レッスンは3区分のシンプルな構成です。

1. **このレッスンでできるようになること** — 練習後の変化が一文で示される
2. **STEP** — 小さなSTEPを1つずつ表示。各STEPが専用の楽譜・ルール・課題を持ち、編集が必要なSTEPには拍グリッドエディタが埋め込まれています
3. **できたかチェック** — チェックボックス式の自己評価

補足(なぜ効くのか・よくある失敗)は折りたたみの「豆知識」に収納。編集STEPは条件と操作課題を満たすまで「次のSTEP」が有効にならず、全STEP完了で「レッスン完了」が解放されます。

**テンポ/キーのはしご**: 多くのレッスンに 60→70→80→100 BPM、C→F→B♭ のはしごが付いています。「クリアした!」で達成を記録し、自分のペースで上がります(進行の条件にはなりません)。**2・4拍クリック**(ジャズの定番メトロノーム)も切り替えられます。

## フレーズ作成(拍グリッドエディタ)

1小節=4拍の**固定の枠**をマスで埋める方式です。合計は構造上つねにぴったり4拍×小節数になるため、拍数の計算や「どこが超過しているか探し」は不要です。

- 各マス: **●鳴らす / ○休む / →のばす**(前の音を延長)。「のばす」で拍・小節線をまたぐシンコペーションも作れます(譜面はタイで表示)
- 拍ごとに分割を切替: **4分 / 8分 / 3連 / 16分**(レッスンごとに許可される分割が変わります)
- 音の高さ: 音符をタップ→**▲▼**で、そのSTEPの使用可能音セット(コードトーン/ガイドトーン/ブルース等)の中を上下移動。度数ラベル(Root/3rdなど)を常時表示
- **アーティキュレーション**: 普通 / アクセント(>) / 短く(スタッカート) / 長く(テヌート)。譜面記号と再生(音量・音価)の両方に反映
- **⧉前の小節をコピー**: モチーフやリフを4小節・12小節へ一気に展開(音は各小節のコードの度数へ自動対応)
- 編集内容とUndo/Redo履歴はSTEPごとに保持。キー変更時は確認のうえ新しいキーで作り直します
- 小節数は課題に応じて4〜12小節。自由練習の「🎼 自分のフレーズを作る」でも同じエディタを使えます(素材: コードトーン/ガイドトーン/ブルース)

**「音を確認」は見本演奏ではありません。** ユーザー自身が編集した楽譜を、設定したBPMと正しい音程で再生して内容を確認するための機能です。

## 楽器選択の仕組み

- **マイ楽器(基本設定)**: ホーム画面で普段使う楽器・譜表・表示ピッチ・(ギターの)譜面表示を保存します
- **今回の練習設定**: レッスンや曲で実践を開いたときに、マイ楽器を初期値とした確認パネルが表示され、**今回だけ**別の楽器や譜面に変更できます(マイ楽器は自動では上書きされません)
- 「この設定をマイ楽器に保存」を選んだときだけ基本設定が更新されます
- そのままでよければ「この内容で始める」1タップで練習を開始できます

## ギター対応(TAB譜)

Guitarを選ぶと、譜面表示を **五線譜 / TAB譜 / 五線譜+TAB**(初期値)から選べます。

- チューニング: Standard(E2 A2 D3 G3 B3 E4)※データとして管理、将来の拡張に対応
- ポジション: 自動 / オープン / 第3・5・7・9ポジション(指定範囲のフレットでTABを生成)
- 開放弦の使用: オン/オフ
- TAB生成は「指定ポジション内で、前の音から近い弦・フレットを選ぶ」方針。五線譜・再生音とTABの音程は常に一致します
- TABのみ表示でも、度数表示をオンにすると Root / 3rd / 7th などの役割ラベルがTAB上に表示されます
- ギターの五線譜は慣習どおり実音より1オクターブ上に記譜(Written選択時)

## 見本演奏を使用しない方針

現在のバージョンでは、**自動生成された見本演奏や見本フレーズは使用していません**。
使用する音、ターゲット、練習条件を確認し、ユーザー自身がフレーズを作ることを重視しています。

**この方針の境界(明文化)**:
- **OK**: メトロノーム/カウントイン/シンプルなコード伴奏/**譜面に表示された固定の教材譜例の再生**(「この音を確認」— リズムパターンやガイドトーンの道など、あらかじめ楽譜として提示されている内容)/**ユーザー自身が編集したフレーズの再生**(「音を確認」「コードと一緒に確認」)
- **NG**: AIや乱数による見本ソロ・見本フレーズの生成、その再生。「お手本を聴いてまねる」形の機能

つまり「譜面に見えているものを音にする」機能だけを持ち、「演奏のお手本」は提供しません。リズム中心の第1章でも、聴くのはすべて譜面どおりの固定譜例です。

## 対応機能

- 練習メニュー: ii–V–I / マイナーii–V–I / ジャズブルース / 循環進行 / 逆循環 / 枯葉進行 / マイナーブルース / 自由入力(実践モード)
- キー変更(12キー、全表示が自動移調)
- BPM設定(40〜220、再生中もライブ反映)
- メトロノーム / 4カウントイン / ループ(全体・2小節・1小節)/ 簡易コード伴奏
- スウィング再生(なし/軽め/標準/強め。16分はイーブン)
- 五線譜表示(VexFlow): 再生位置ハイライト、3連符対応、音名・ドレミ・度数の補助表示
- コード情報(コードトーン・ガイドトーン・おすすめスケール・テンション・度数)
- 楽器設定: Piano / Guitar / Bass / Trumpet / Trombone / Soprano・Alto・Tenor・Baritone Sax / Vocal / Other
- 移調楽器対応(Concert / Written 切替。B♭管・E♭管の記譜表示、ギターの記譜オクターブ)
- ギターTAB譜(下記参照)
- 練習コースの進捗保存・自己評価(localStorage)
- 練習ログ(日時・メニュー・キー・BPM・楽器・課題・メモ)
- 表示言語: 日本語 / English

※録音機能は未実装です(今後の方向性を参照)。

## 技術構成

- React 18 + TypeScript 5 + Vite 5
- Tone.js 15(メトロノーム・伴奏などの音声再生)
- VexFlow 4(五線譜レンダリング)
- データ保存: localStorage(保存層は `src/state/storage.ts` に抽象化)

```
src/
├── theory/       # 音楽理論ロジック(音名・コード・進行・移調・ノート生成)
├── audio/        # Tone.js 再生エンジン
├── data/         # 練習コンテンツ(コース・レッスン・課題・ミッション)※UIから分離
├── hooks/        # usePracticePlayback(再生制御の共有フック)
├── screens/      # ホーム / 自由練習 / 練習コース / 曲で実践
├── components/   # 五線譜・進行表示・コード情報・練習ログ
├── state/        # 進捗保存の抽象化
└── i18n.ts       # 日英表示文字列
```

## ローカルでの起動方法

```bash
git clone https://github.com/Tanbo-hiroooi/Jazz-improvisation.git
cd Jazz-improvisation
npm install
npm run dev      # http://localhost:5173
```

## ビルド

```bash
npm run build      # 型チェック + 本番ビルド(dist/ に出力)
npm run preview    # 本番ビルドの確認
npm run typecheck  # 型チェックのみ
```

## デプロイ

`main` ブランチへの push で、GitHub Actions([.github/workflows/deploy.yml](.github/workflows/deploy.yml))が自動でビルドし GitHub Pages へデプロイします。手動実行も Actions タブの「Run workflow」から可能です。

## iOSでの利用について

- 音は最初のタップ(Startなど)で有効になります(AudioContextの制約に対応済み)
- マナーモード(無音スイッチ)ONだと音が出ないことがあります
- 将来的なiOSアプリ化は、ロジック層とUI層を分離した現構成のままCapacitorでラップする経路を想定しています
