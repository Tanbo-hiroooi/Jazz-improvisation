# Jazz Phrase Lab

楽器を演奏でき、楽譜も読める人が、コード進行を理解しながら**自分でジャズのアドリブを作る**ための練習Webアプリです。

「コードを確認 → 使える音とルールを確認 → 伴奏に合わせて自分で演奏 → 自己評価」のサイクルを繰り返し、譜面演奏から即興演奏への橋渡しをします。

公開URL: https://tanbo-hiroooi.github.io/Jazz-improvisation/

## 想定ユーザー

- 楽器の演奏経験がある(吹奏楽・クラシック・ポップスなど)
- 五線譜が読める
- アドリブは初心者
- スケールは分かるが、フレーズにできない
- コード進行に合わせた音選びを学びたい

## 3つの主要モード

### 自由練習
苦手な技術を自分で選び、反復練習するモード。コードトーン/ガイドトーン/アプローチ/ターゲット/スケールの表示を切り替え、11種のリズムパターンや練習課題(ルール提示型)と組み合わせて練習します。

### 練習コース
「なぜこの練習をするのか」を理解しながら、段階的に進めるモード。全6章・18レッスンの「アドリブ入門コース」を収録しています。

### 曲で実践
覚えた技術を、ジャズブルースや長いコード進行の中で使用するモード。実践ミッション(例:「1コーラス目はコードトーンだけ」)を選び、コーラス周回カウントを見ながら1コーラスのソロに挑戦します。

## 練習コースの特徴

各レッスンは次の流れで構成されています。

1. **こんな悩みを解決します** — 演奏上の悩みからレッスンが始まる
2. **今日できるようになること** — 練習後の変化が一文で示される
3. **なぜ、この練習をするの?** — 練習理由と実際に使う場面(折りたたみ)
4. **今回使うコード進行と音** — 五線譜・音名・度数で表示
5. **練習ルールとSTEP** — 2〜3個の小さなステップに分割、現在位置を表示
6. **うまくできているサイン** — 成功の目安を自分で判断できる
7. **よくある失敗** — 不安にならないための失敗例と対処
8. **今日のチェック** — チェックボックス式の自己評価
9. **次のレッスンとのつながり** — 学びがどう発展するか

章構成: ①コードに合う音を選ぶ → ②コードの変化を音で表す → ③次のコードへ自然に着地する → ④音をフレーズに変える → ⑤短いアイデアを発展させる → ⑥コード進行の中でまとめる

難しいレッスンは「自由練習で復習する」からキー・BPM・進行・表示内容を引き継いで反復でき、終わったら元のレッスンに戻れます。

## 主な練習項目

コードトーン / ガイドトーン / ターゲットノート / アプローチノート(半音・エンクロージャー)/ リズム(スウィング8分・裏拍・3連符・チャールストンほか)/ 休符・音数制限 / モチーフ展開 / ii–V–I / ジャズブルース / 曲形式での実践

## 見本演奏を使用しない方針

現在のバージョンでは、**自動生成された見本演奏や見本フレーズは使用していません**。
使用する音、ターゲット、練習条件を確認し、ユーザー自身がフレーズを作ることを重視しています。

音声機能は、メトロノーム/カウントイン/シンプルなコード伴奏/表示中の音の確認(「音を確認」)/リズムのみの視覚ガイドに限定しています。

## 対応機能

- 練習メニュー: ii–V–I / マイナーii–V–I / ジャズブルース / 循環進行 / 逆循環 / 枯葉進行 / マイナーブルース / 自由入力(実践モード)
- キー変更(12キー、全表示が自動移調)
- BPM設定(40〜220、再生中もライブ反映)
- メトロノーム / 4カウントイン / ループ(全体・2小節・1小節)/ 簡易コード伴奏
- スウィング再生(なし/軽め/標準/強め。16分はイーブン)
- 五線譜表示(VexFlow): 再生位置ハイライト、3連符対応、音名・ドレミ・度数の補助表示
- コード情報(コードトーン・ガイドトーン・おすすめスケール・テンション・度数)
- 楽器設定: Piano / Guitar / Bass / Trumpet / Trombone / Soprano・Alto・Tenor・Baritone Sax / Vocal / Other
- 移調楽器対応(Concert / Written 切替。B♭管・E♭管の記譜表示)
- 練習コースの進捗保存・自己評価(localStorage)
- 練習ログ(日時・メニュー・キー・BPM・楽器・課題・メモ)
- 表示言語: 日本語 / English

※録音機能は未実装です(今後の方向性を参照)。

## 技術構成

- React 18 + TypeScript 5 + Vite 5
- Tone.js 15(メトロノーム・伴奏などの音声再生)
- VexFlow 4(五線譜レンダリング)
- データ保存: localStorage(保存層は `src/state/storage.ts` に抽象化)

```
src/
├── theory/       # 音楽理論ロジック(音名・コード・進行・移調・ノート生成)
├── audio/        # Tone.js 再生エンジン
├── data/         # 練習コンテンツ(コース・レッスン・課題・ミッション)※UIから分離
├── hooks/        # usePracticePlayback(再生制御の共有フック)
├── screens/      # ホーム / 自由練習 / 練習コース / 曲で実践
├── components/   # 五線譜・進行表示・コード情報・練習ログ
├── state/        # 進捗保存の抽象化
└── i18n.ts       # 日英表示文字列
```

## ローカルでの起動方法

```bash
git clone https://github.com/Tanbo-hiroooi/Jazz-improvisation.git
cd Jazz-improvisation
npm install
npm run dev      # http://localhost:5173
```

## ビルド

```bash
npm run build      # 型チェック + 本番ビルド(dist/ に出力)
npm run preview    # 本番ビルドの確認
npm run typecheck  # 型チェックのみ
```

## デプロイ

`main` ブランチへの push で、GitHub Actions([.github/workflows/deploy.yml](.github/workflows/deploy.yml))が自動でビルドし GitHub Pages へデプロイします。手動実行も Actions タブの「Run workflow」から可能です。

## iOSでの利用について

- 音は最初のタップ(Startなど)で有効になります(AudioContextの制約に対応済み)
- マナーモード(無音スイッチ)ONだと音が出ないことがあります
- 将来的なiOSアプリ化は、ロジック層とUI層を分離した現構成のままCapacitorでラップする経路を想定しています
